# AGENTS.md

Content repository for [karepker.com](https://karepker.com), Kar Epker's personal
website. This repo holds only content and site configuration; it is built by an
external Python toolchain.

## Build system

The site was originally a Jekyll site and is now built by a custom Python
harness that lives **outside this repo** at `../markdown_webharness/` (see its
`CLAUDE.md`). The webzone-specific build code is in
`../markdown_webharness/build/sites/webzone/`. That code, not Jekyll, is the
source of truth for how content here is rendered.

Build output goes to `_site/` (gitignored). Deployment uses a `gh-pages` git
worktree checked out at `_site/`.

## Layout

Consumed by the harness:

- `_posts/*.md` â€” blog posts, named `YYYY-MM-DD-slug.md`. Frontmatter uses
  `title`, `image` (cover), `image_title`; optional `subtitle`, `author`,
  `front_page`. A `<!--more-->` marker splits the homepage excerpt from the
  rest. Rendered to `_site/<slug>/index.html`.
- `about.md`, `projects.md`, `recommendations.md` â€” top-level nav pages
  (`ROOT_PAGE_FILES` in the harness's `config.py`). To add a nav page, add the
  file here **and** register it there.
- `static/` â€” `site.scss` (compiled to `site.css`), `syntax.css`, and the two
  icon PNGs. Only these files are picked up.
- `media/` â€” post images (gitignored; see Media below).
- `CNAME` â€” copied verbatim to the output root.

The homepage (`index.html`), `404.html`, and `feed.xml` are **generated by the
harness** (`layout.py` / `feed.py`), not from files here. The old Jekyll
templates for these (plus `_layouts/` and `_includes/`) have been removed; the
equivalents now live in the harness's `layout.py` / `feed.py`.

Local-only, gitignored: `_site/`, `_drafts/` (in-progress posts, same format as
`_posts/`), `_reference/` (research materials), and `media/`.

## Utility scripts (`_util/`)

Run from the repo root.

- `build.sh` â€” builds the site into `_site/` via
  `build_website --site webzone` (installed from the harness at
  `~/.local/bin/build_website`).
- `deploy.sh` â€” refuses to run with a dirty tree; ensures `_site/` is a
  `gh-pages` worktree, rebuilds, adds `.nojekyll`, commits, and pushes
  `gh-pages` (served by GitHub Pages).
- `optimize_images.sh DIR` â€” resizes JPEGs to max 1200px, strips metadata,
  stamps a JPEG-comment marker so re-runs are idempotent. `-f` forces; a
  `keep-original` comment opts an image out entirely.
- `upload_media.sh` â€” syncs `media/` to Backblaze B2 (`b2://karepker-com/`).
- `replace_media_with_cdn.sh FILE` â€” rewrites `/media/...` references in a post
  to `https://media.karepker.com/file/karepker-com/...` (CDN).

## Conventions

- End the first sentence of commit messages with a period.
- Posts and pages are Markdown with YAML frontmatter (Kramdown-style extensions
  supported by the harness's `render/` component).
- Images are served from a CDN (`media.karepker.com`, backed by B2); posts
  reference full CDN URLs. Optimize with `optimize_images.sh` and upload with
  `upload_media.sh` before referencing.
